# Integration Plan

**Project:** AI Recruitment Assistant  
**Version:** 1.0  
**Date:** April 27, 2026  
**Owner:** @integration.eng

---

## 1. Overview

This document outlines the integration approach for connecting the Next.js frontend with the FastAPI + CrewAI backend. The goal is to ensure end-to-end data flow from user input (job description) through the agent pipeline to candidate recommendations.

---

## 2. Frontend-Backend Integration

### 2.1 Architecture

```
┌─────────────────────┐     ┌─────────────────────┐
│   Frontend (Next.js)│     │ Backend (FastAPI)   │
│   Port: 3000        │────▶│ Port: 8000          │
│                     │     │                     │
│  - ChatInterface    │     │  - /api/recruit      │
│  - ResultsList      │     │  - /api/health       │
│  - CandidateCard    │     │  - CrewAI Pipeline   │
└─────────────────────┘     └─────────────────────┘
```

### 2.2 Communication

| Component | Protocol | Format |
|-----------|----------|--------|
| Frontend → Backend | HTTP POST | JSON |
| Backend → Frontend | HTTP Response | JSON |
| API Base URL | `http://localhost:8000` | - |

## 3. API Connection Setup

### 3.1 Endpoints

| Endpoint | Method | Purpose | Status |
|----------|--------|---------|--------|
| `/api/recruit` | POST | Submit job description, get recommendations | 🔶 Needs Fix |
| `/api/health` | GET | Check backend availability | ✅ Ready |
| `/api/candidates` | GET | List cached candidates | 🔶 Partial |

### 3.2 Request Format

```typescript
// Frontend Request (frontend/lib/api.ts)
interface RecruitmentRequest {
  job_description: string;
  max_candidates?: number;
}
```

```python
# Backend Request (backend/app/models.py)
class RecruitmentRequest(BaseModel):
    job_description: str
    max_candidates: int = 10
```

### 3.3 Response Format

```typescript
// Frontend expects (frontend/lib/api.ts)
interface RecruitmentResponse {
  candidates: EvaluatedCandidate[];
  total_searched: number;
  processing_time_seconds: number;
}
```

```python
# Backend returns (backend/app/models.py)
class RecruitmentResponse(BaseModel):
    job_description: str
    recommendations: RankedRecommendation
    status: str = "completed"
```

## 4. Issues Identified

### 4.1 Response Format Mismatch

| Issue | Frontend Expects | Backend Returns |
|-------|------------------|-----------------|
| Root field | `candidates` | `recommendations.candidates` |
| Root field | `total_searched` | `recommendations.total_searched` |
| Root field | `processing_time_seconds` | `recommendations.processing_time_seconds` |

### 4.2 Crew Execution Incomplete

The `run_recruitment_pipeline` function in `crew.py` is incomplete - it creates the crew but doesn't execute it or parse results properly.
